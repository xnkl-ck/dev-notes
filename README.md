## 🎯 使用说明

> 💡 **不知道如何记录？** 查看 [📝 记录指南](./HOW_TO_RECORD.md)，里面有详细的决策树和示例！

### 记录格式
每个知识点建议包含：
1. **问题描述/场景** - 什么情况下遇到的
2. **错误示例** - 问题代码（如果有）
3. **正确做法** - 解决方案
4. **原理分析** - 为什么会这样
5. **总结与延伸** - 举一反三

### 标签系统
使用标签方便后续搜索：
- `#react` `#vue` `#javascript` - 技术栈
- `#bugfix` `#optimization` `#refactor` - 问题类型
- `#performance` `#security` - 关注点
