# **普通函数动态参数和箭头函数的动态参数有什么区别？**

## 前言

JavaScript 函数处理参数的能力非常灵活，其中“动态参数”是指函数接收不确定数量的参数。普通函数和箭头函数在动态参数处理上有显著区别，特别是在 `arguments` 对象的访问和默认参数方面。掌握这两者的不同，能帮助我们在编写灵活函数时避免误用，提高代码的正确性和性能。

## 动态参数概述

动态参数通常指函数能够接受不固定数量的参数。传统 JavaScript 函数中，可以通过 `arguments` 对象访问所有传入参数，而 ES6 引入的剩余参数（rest parameters）更方便地收集参数。

## 普通函数中的动态参数

普通函数中，`arguments` 是一个类数组对象，包含所有传入参数，无论函数定义时有多少参数。

```plain
function sum() {
  let total = 0;
  for(let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return total;
}
console.log(sum(1, 2, 3)); // 6
```

普通函数可以使用剩余参数语法，效果更好：

```plain
function sum(...nums) {
  return nums.reduce((acc, val) => acc + val, 0);
}
```

## 箭头函数中的动态参数

箭头函数没有自己的 `arguments` 对象，访问 `arguments` 实际上是访问定义时的外层函数的 `arguments`。

```plain
function outer() {
  const arrow = () => {
    console.log(arguments);
  }
  arrow(1, 2); // 输出 outer 函数的 arguments
}
outer(3, 4);
```

箭头函数需要使用剩余参数来获取动态参数。

```plain
const arrowSum = (...nums) => nums.reduce((a, b) => a + b, 0);
console.log(arrowSum(1, 2, 3)); // 6
```

## 主要区别总结

- 普通函数有自己的 `arguments` 对象，可以直接访问所有参数。
- 箭头函数没有自己的 `arguments`，访问的是外层作用域的 `arguments`。
- 两者都可以使用剩余参数语法来处理动态参数，但箭头函数必须使用剩余参数才能访问所有参数。

## 面试回答

**问题：普通函数的动态参数和箭头函数的动态参数有什么区别？**

回答示范：

普通函数拥有自己的 `arguments` 对象，可以通过它访问所有传入参数，而箭头函数没有自己的 `arguments`，访问的其实是外层函数的 `arguments`。因此，箭头函数通常通过剩余参数语法来获取动态参数。两者在动态参数访问上存在本质区别，了解这些有助于避免在箭头函数中错误使用 `arguments`，保证代码正确性。

## 总结

理解普通函数和箭头函数在动态参数处理上的区别，是编写灵活函数和避免潜在错误的关键。箭头函数没有自己的 `arguments`，需要使用剩余参数，而普通函数可直接使用 `arguments`。合理选择和运用这两种方式，能提高代码的可读性和稳定性。
