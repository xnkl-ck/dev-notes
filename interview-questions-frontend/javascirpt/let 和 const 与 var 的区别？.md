# **let 和 const 与 var 的区别？**

##  前言

JavaScript 中声明变量主要用到 `var`、`let` 和 `const`，它们在作用域、提升机制、可修改性等方面有明显区别。理解这些区别有助于写出更安全、易维护的代码。本文详细解析三者的核心差异。

## var 的特点

`var` 声明的变量具有函数作用域或全局作用域，不具备块级作用域。变量声明会被提升（Hoisting），即声明提前到作用域顶部，但赋值不提升，访问声明前变量值为 `undefined`。同一作用域内可以多次声明同名变量，不会报错。变量值可被重新赋值。

示例：

```plain
console.log(a); // undefined
var a = 10;

if (true) {
  var b = 20;
}
console.log(b); // 20，var 没有块级作用域
```

## let 的特点

`let` 声明的变量具有块级作用域，只在声明所在块或语句中有效。在声明前访问会触发暂时性死区（Temporal Dead Zone），抛出引用错误（ReferenceError）。不能在同一作用域内重复声明同名变量。变量值可变。

示例：

```plain
{
  let c = 30;
  console.log(c); // 30
}
// console.log(c); // 报错，c 不存在于此作用域
```

## const 的特点

`const` 也具有块级作用域，声明时必须初始化，且绑定的变量不可重新赋值。对于引用类型，绑定地址不可变，但对象内部属性仍可修改。不能重复声明。

示例：

```plain
const d = 40;
// d = 50; // 报错，const 绑定不可变

const obj = { name: 'Alice' };
obj.name = 'Bob'; // 合法，修改对象属性
```

## 三者区别总结

`var` 有函数作用域，存在变量提升和重复声明的风险；`let` 和 `const` 有块级作用域，消除了提升的隐患和重复声明问题，更安全可靠。`const` 用于声明常量绑定，提高代码的稳定性。

## 面试回答示范

**问题：请说说 let、const 和 var 的区别？**

回答示例：

“`var` 具有函数作用域，声明会被提升，访问声明前变量值为 `undefined`，允许重复声明。`let` 和 `const` 是 ES6 新增，具有块级作用域，访问声明前变量会报错，不能重复声明。`let` 变量可重新赋值，`const` 声明的变量必须初始化且绑定不可更改，但引用类型的内部属性可以改。`let` 和 `const` 更安全，推荐使用。”

------

## 总结

掌握 `var`、`let` 和 `const` 的差异，能有效避免作用域混乱和变量提升问题，提高代码质量，是现代 JavaScript 开发的基础。
