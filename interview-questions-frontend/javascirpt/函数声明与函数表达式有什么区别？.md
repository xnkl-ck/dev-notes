# **函数声明与函数表达式有什么区别？**

## 前言

JavaScript 中定义函数的方式多样，最常见的是函数声明和函数表达式。它们在语法、执行时机、作用域和提升机制等方面存在差异。理解这两者的区别，有助于避免潜在的代码错误，提高代码的可读性和维护性。

## 函数声明与函数表达式的基本定义

- **函数声明**：使用 `function` 关键字定义的具名函数，直接声明在代码中。

```plain
function greet() {
  console.log('Hello');
}
```

- **函数表达式**：将函数赋值给变量的表达式，函数可以是匿名或具名。

```plain
const greet = function() {
  console.log('Hello');
};
```

## 主要区别

### 1. 提升（Hoisting）

函数声明会被提升，意味着函数可以在声明之前调用。

```plain
sayHello(); // 输出 "Hello"

function sayHello() {
  console.log('Hello');
}
```

函数表达式不会提升，调用必须在定义之后，否则报错。

```plain
sayHello(); // 报错：sayHello is not a function

const sayHello = function() {
  console.log('Hello');
};
```

### 2. 变量赋值时间

函数表达式是变量赋值的一部分，只有运行到赋值语句时，变量才指向函数。

函数声明在代码编译阶段就完成函数对象的创建。

### 3. 命名

函数声明必须有函数名。

函数表达式可以是匿名函数，也可以是具名函数。

```plain
const foo = function bar() {
  // 这里 bar 是函数名，仅用于函数内部递归
};
```

### 4. 作用域和调用方式

函数声明通常定义在全局或函数作用域中。

函数表达式常用于回调、闭包、条件定义等灵活场景。

### 5. 调试体验

函数表达式的匿名函数调试时堆栈信息不如具名函数清晰，建议为表达式赋予名称以便调试。

## 适用场景差异

- 函数声明适合定义独立的功能块，便于阅读和调用。
- 函数表达式适合动态赋值、作为回调或闭包，便于灵活控制执行时机。

## 面试回答

**问题：函数声明和函数表达式有什么区别？**

回答示范：

函数声明是在代码编译阶段完成提升，可以在声明之前调用，具有函数名；函数表达式是将函数赋值给变量，变量提升但赋值不提升，调用必须在赋值之后。函数表达式可以是匿名函数，常用于回调或动态定义。两者在提升机制、调用时机和灵活性上存在明显差异，合理选择能提高代码健壮性和可维护性。

## 总结

函数声明与函数表达式是 JavaScript 中两种常见且重要的函数定义方式。了解它们的区别，特别是提升行为和调用时机，是避免运行错误和优化代码结构的基础。掌握这两者的使用场景，将使代码更加规范和高效。
