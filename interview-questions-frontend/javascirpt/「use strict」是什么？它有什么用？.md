# **「use strict」是什么？它有什么用？**

## 前言

JavaScript 是一门设计自由度极高的语言，这种灵活性带来了快速开发的便利，也引入了诸多潜在问题。为了提升代码的健壮性与安全性，ECMAScript 5 引入了“严格模式”——即 `"use strict"`。这是一种以更严格语法规则运行 JavaScript 代码的方式，可以帮助开发者规避常见陷阱、提前发现问题。本文将深入解析 `use strict` 的作用、特性及使用方式。

## 正文

### 一、什么是 "use strict"

`"use strict"` 是一种**启用严格模式的声明字符串**，作用范围可以是**整个脚本**或**单个函数**。一旦启用，JavaScript 引擎会按照更严格、更安全的方式来执行代码，抛出一些非严格模式下不会报错的问题。

### 二、严格模式的作用与限制

以下是严格模式下的一些关键变化和增强行为：

1. **禁止使用未声明的变量**

非严格模式下，写 `a = 10` 会隐式创建一个全局变量。而在严格模式中，必须先通过 `var` / `let` / `const` 声明变量，否则抛出 `ReferenceError`。

1. **禁止删除变量或函数**

如 `delete a`、`delete functionName` 在严格模式下将报错。

1. **禁止重复声明参数名**

如 `function test(a, a)` 是合法的非严格代码，但在严格模式下是语法错误。

1. `this` **的默认绑定更安全**

在严格模式中，未指定上下文调用函数时 `this` 不再自动指向 `window`，而是 `undefined`。例如：

```javascript
function foo() {
  console.log(this);
}
foo(); // 非严格模式下打印 window，严格模式下打印 undefined
```

1. **禁止八进制语法**

如 `var num = 010`（代表十进制 8）在严格模式下将报错。

1. **限制对保留字的使用**

如 `package`、`interface` 等未来保留字不能被用作变量名。

1. **限制函数内部使用** `arguments.callee` **和** `arguments.caller`

这些写法在严格模式中被禁用，有助于优化和提升代码可维护性。

### 三、如何启用严格模式

- 脚本级别启用：

```javascript
"use strict";
x = 1; // 报错：未声明变量
```

- 函数级别启用：

```javascript
function test() {
  "use strict";
  y = 2; // 报错：未声明变量
}
```

注意：`"use strict"` 必须写在作用域的最顶部，且是字面量字符串，不能用变量或拼接的方式引入。

## 面试回答示范

**问题：JavaScript 中的** `"use strict"` **有什么作用？为什么要用它？**

回答示例：

“`use strict` 是 ES5 引入的一种严格模式声明，可以让 JavaScript 在更严格、更安全的环境下运行。它主要用于捕捉隐式错误，比如禁止使用未声明变量、禁止使用重复参数名、禁止对保留字赋值等。

在严格模式下，代码执行会更加规范，有助于提升安全性和可维护性，同时也方便 JavaScript 引擎做性能优化。我在项目中会将严格模式作为默认写法放在文件或函数顶部。”

------

## 总结

严格模式是 JavaScript 为了提升语言健壮性而引入的重要特性。通过 `"use strict"`，我们可以提前暴露潜在的运行时问题，强制开发者遵循更规范的编码习惯。虽然现代模块化系统（如 ES modules）默认启用严格模式，但深入理解其作用依然是掌握 JavaScript 基础的重要一环。
