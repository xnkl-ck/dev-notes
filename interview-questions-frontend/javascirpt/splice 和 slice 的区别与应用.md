# **splice 和 slice 的区别与应用**

## 前言

在 JavaScript 中，`splice` 和 `slice` 是数组操作中两个非常常用的方法。它们的名称相似，但功能和行为有很大不同。理解它们的区别不仅能帮助你避免常见的错误，还能让你的数组操作更高效和准确。本文将详细对比 `splice` 和 `slice`，解析它们的用法、返回值、对原数组的影响以及典型应用场景。

## splice 与 slice：功能和用法对比

### splice 方法

`splice` 方法用于修改数组本身，可以从数组中添加、删除或替换元素。其语法为：

```plain
array.splice(start, deleteCount, item1, item2, ...)
```

- `start`：起始位置（负数表示从末尾开始计数）。
- `deleteCount`：删除的元素个数。
- 后续参数（可选）：用于插入的新元素。

`splice` 会返回被删除的元素组成的新数组，且原数组会被修改。

示例：

```plain
const arr = [1, 2, 3, 4, 5];
const removed = arr.splice(1, 2, 'a', 'b');
console.log(arr); // [1, 'a', 'b', 4, 5]
console.log(removed); // [2, 3]
```

### slice 方法

`slice` 方法用于从数组中提取一部分，返回一个新数组，不修改原数组。其语法为：

```plain
array.slice(start, end)
```

- `start`：提取开始位置（包含该索引）。
- `end`（可选）：提取结束位置（不包含该索引），默认到数组末尾。

示例：

```plain
const arr = [1, 2, 3, 4, 5];
const newArr = arr.slice(1, 4);
console.log(newArr); // [2, 3, 4]
console.log(arr); // [1, 2, 3, 4, 5] 原数组未变
```

### 主要区别总结

- **是否修改原数组**：`splice` 会修改原数组，`slice` 不修改。
- **功能定位**：`splice` 是增删改的综合方法，`slice` 是截取子数组。
- **返回值**：`splice` 返回被删除的元素数组，`slice` 返回截取的新数组。
- **参数意义**：`splice` 第二个参数是删除元素数量，`slice` 第二个参数是结束位置索引。

## 何时使用 splice，何时使用 slice

- 需要**修改原数组**，增删改元素时用 `splice`。
- 需要**获取数组的一部分副本**，不改变原数组时用 `slice`。
- 在函数参数传递或链式操作中，为了避免副作用，通常用 `slice` 创建新数组。

## 面试回答示范

问：“splice 和 slice 有什么区别？它们各自的作用和适用场景是什么？”

“`splice` 和 `slice` 都是数组常用方法，但它们的作用和行为差别很大。`splice` 用于修改原数组，可以删除、替换或者插入元素。它接收起始位置和删除元素个数，还可以传入新元素，返回被删除的元素数组，操作后数组长度和内容都会改变。

而 `slice` 是用来截取数组的一部分，返回一个新的数组，**不改变原数组**。它的参数是开始索引和结束索引（不包含），可以用来获取数组的子集或者克隆数组。

简单来说，如果想在原数组上做增删改，就用 `splice`；如果想要拿一段数据做处理但不影响原数组，用 `slice`。

在开发中，考虑到副作用和代码安全，常用 `slice` 复制数组避免修改原数组，而 `splice` 则适合需要直接操作原数组的场景。”

## 总结

`splice` 和 `slice` 是操作数组时最基础的两个工具。`splice` 是修改型操作，功能强大但有副作用；`slice` 是非修改型操作，适合提取和复制数组。掌握它们的区别和用法，是避免数组操作错误、提升代码质量的重要基础。
